# MediaPipe FaceLandmarker — 랜드마크와 컨투어

## MediaPipe FaceLandmarker란

MediaPipe FaceLandmarker는 Google이 만든 얼굴 랜드마크 감지 모델이다. 브라우저에서 WebAssembly + GPU로 직접 실행되며, 서버 없이 클라이언트에서 추론이 완료된다.

입력: 이미지 또는 영상 프레임  
출력: 478개 랜드마크의 (x, y, z) 좌표. x, y는 0~1 사이 정규화 값 (이미지 너비/높이 대비 비율).

### 왜 478개인가

기본 얼굴 메시는 468개 포인트다. 여기에 왼쪽/오른쪽 홍채 각각 5개씩 추가되어 478개가 된다.

- 0~467: 얼굴 메시 (피부, 눈썹, 코, 입술, 눈 개구부, 얼굴 외곽)
- 468~472: 오른쪽 홍채 (중심, 우, 하, 좌, 상)
- 473~477: 왼쪽 홍채 (중심, 우, 하, 좌, 상)

---

## 컨투어(Contour)란

MediaPipe는 랜드마크 간의 연결 정보를 `FACE_LANDMARKS_*` 상수로 제공한다. 이 상수들은 `{ start: number, end: number }` 엣지의 배열로 구성되어 있다.

문제는 이 엣지 배열이 **순서가 보장된 폴리곤이 아니라**, 인접 그래프 형태의 연결 목록이라는 점이다. 따라서 Canvas의 `beginPath → moveTo → lineTo → closePath → fill`로 직접 사용할 수 없다.

이 프로젝트에서는 각 영역의 컨투어를 **직접 디버그해서 올바른 순서로 수동 정렬**했다.

---

## 컨투어 인덱스 배열 (검증 완료)

각 배열은 Canvas `fill()` 시 올바른 닫힌 폴리곤을 만드는 순서로 정렬되어 있다. 아래 인덱스들은 디버그 dot 시각화로 직접 확인했다.

### 눈 개구부

```ts
const RIGHT_EYE_CONTOUR = [
  33, 246, 161, 160, 159, 158, 157, 173, 133, 155, 154, 153, 145, 144, 163, 7,
];
const LEFT_EYE_CONTOUR = [
  263, 466, 388, 387, 386, 385, 384, 398, 362, 382, 381, 380, 374, 373, 390, 249,
];
```

위 눈꺼풀(33→133) + 아래 눈꺼풀(133→33) 순서로 닫힌 폴리곤. 눈 개구부(검은자+흰자가 보이는 열린 공간)만을 감싼다.

### 눈썹

```ts
const RIGHT_EYEBROW_CONTOUR = [46, 53, 52, 65, 55, 107, 66, 105, 63, 70];
const LEFT_EYEBROW_CONTOUR  = [276, 283, 282, 295, 285, 336, 296, 334, 293, 300];
```

- 하단 라인 (안쪽→바깥): 46→53→52→65→55
- 상단 라인 (바깥→안쪽): 107→66→105→63→70
- MediaPipe 눈썹은 10포인트뿐이라 세밀하지 않음 — 모델 한계

### 입술

```ts
const LIPS_CONTOUR = [
  61, 185, 40, 39, 37, 0, 267, 269, 270, 409,
  291, 375, 321, 405, 314, 17, 84, 181, 91, 146,
];
```

`FACE_LANDMARKS_LIPS`에는 외곽 루프(61↔291)와 내곽 루프(78↔308)가 섞여 있다. 외곽 루프만 수동 추출했다.

- 위 외곽: 61→185→40→39→37→0→267→269→270→409→291
- 아래 외곽: 291→375→321→405→314→17→84→181→91→146→(61로 닫힘)

### 얼굴 외곽

```ts
const FACE_OVAL_CONTOUR = [
  10, 338, 297, 332, 284, 251, 389, 356, 454, 323, 361, 288, 397, 365, 379, 378,
  400, 377, 152, 148, 176, 149, 150, 136, 172, 58, 132, 93, 234, 127, 162, 21,
  54, 103, 67, 109,
];
```

FACE_OVAL 단일 루프. 콘솔에서 실제 엣지 순서를 확인해 정렬했다.

**한계**: 정면 얼굴에서는 잘 동작하지만, 측면 얼굴에서는 얼굴 외곽이 찌그러진 타원으로 나온다. MediaPipe 모델이 측면 얼굴에서 FACE_OVAL을 항상 완전한 타원으로 반환하는 모델 수준의 한계다.

### 코

```ts
const NOSE_CONTOUR = [
  168, 6, 197, 195, 5, 4,
  45, 220, 218, 115, 131, 48, 64, 98, 97,
  2,
  326, 327, 358, 279, 360, 344, 440, 275,
];
```

Tesselation 인접 그래프에서 코 외곽 경로를 추출했다:
- 콧대: 168→6→197→195→5→4
- 코끝→우 콧볼: 4→45→220→218→115→131→48→64→98→97
- 인중 위: 97→2
- 좌 콧볼→코끝: 2→326→327→358→279→360→344→440→275
- 275→4 (closePath로 닫힘)

## 좌표계 변환

MediaPipe는 0~1 정규화 좌표를 반환한다. 실제 픽셀 좌표로 변환할 때:

```ts
const x = landmark.x * imageWidth;
const y = landmark.y * imageHeight;
```

이 프로젝트에서는 두 가지 해상도가 존재한다:

- **analysis 해상도**: MediaPipe 추론에 사용. `max(width, height) <= 600px`으로 축소. 마스크 생성에 사용.
- **display 해상도**: 원본 이미지 크기. Canvas 렌더링에 사용.

랜드마크 좌표는 0~1 정규화값이므로 두 해상도 모두에 적용 가능하다. 단, 마스크 픽셀을 display 좌표로 변환할 때는 `scaleX = displayWidth / analysisWidth`, `scaleY = displayHeight / analysisHeight`를 곱해야 한다.
